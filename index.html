get_data.get_market_range_coingecko(coin_id, start, end)¶

Granularity is automatically determined by Coingecko using the following specifications:

5-minute intervals: 1 day from query time hourly intervals: 1-90 days from query time daily intervals: More than 90 days from query time

Use unix UTC for start and end

Parameters

  • start (int) – The starting time in the range (UTC Unix Timestamp)

  • end (int) – The ending time in the range (UTC Unix Timestamp)

Returns

Returns spot price/volume data in json format if coin_id is valid. Otherwise, None is returned.

Return type

json

get_data.get_ohlc_coingecko(id, vs_currency, days)¶

Coingecko’s OHLC API for OHLC data. The time step intervals are automatically set by Coingecko as follows:

1-2 days: 30 minute intervals 3 < days < 30: 4 hour intervals days > 31: 4 day intervals

Parameters

  • id (str) – The Coingecko coin ID (ethereum, litecoin, etc.)

  • vs_currency (str) – The currency to weigh the coin against (usd, eur, etc.)

Returns

Timestamp (ms), Open, High, Low, Close in a numpy array

get_data.get_ohlc_kraken(pair, days, interval=30)¶

Retrieve OHLC that ranges from a specified date to current. The granularity can be supplied as well. This one might be preferable to the user since it is quite a bit more flexible with time step intervals. :param str pair: This is a ticker pair, such as ETHUSD, XRPUSD, BTCETH, etc. :param int days: Number of days to go back to :type days: integer :param int interval: An integer which specifies the OHLC time interval in

minutes. Valid values are 1, 5, 15, 30, 60, 240, 1440, 10080, 21600

Parameters

kraken (str) – The external source of the data

Returns

a numpy array of volume (USD) and OHLC data; or None if the request cannot be completed either because an invalid symbol pair was passed or for some other reason. All fields are converted to np.float32 type
